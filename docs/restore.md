# restore

Detailed restore workflow for this repository.

## Goal

Restore a full home directory from restic, then re-apply workstation state:

- layered `rpm-ostree` packages
- Flatpak apps
- VS Code extensions

The restore entrypoint is `restore/bootstrap.sh`.

## Prerequisites

- Fedora Silverblue installed
- Network and S3 access available
- `~/.config/restic/password` is present
- `~/.config/restic/env` is present
- repository cloned to `~/.local/share/backup`
- required tools installed (`restic`, `resticprofile`, `jq`)

Important: Keep a copy of `~/.config/restic/password` on a different machine or secure offline storage. Without it, restore is not possible after machine loss.

## Safety notes

- Run full restore from a TTY (for example `Ctrl`+`Alt`+`F3`) or before first graphical login.
- Restoring over a live desktop session can cause process/file conflicts.
- `full-restore` uses `rsync --delete` to make `$HOME` match the selected snapshot.

## Bootstrap script commands

Show help:

```bash
bash ~/.local/share/backup/restore/bootstrap.sh --help
```

Restore home and apply state (default command):

```bash
bash ~/.local/share/backup/restore/bootstrap.sh full-restore
```

Restore a specific snapshot and apply state:

```bash
bash ~/.local/share/backup/restore/bootstrap.sh full-restore <snapshot-id>
```

Restore only home files:

```bash
bash ~/.local/share/backup/restore/bootstrap.sh restore-home
```

Apply only generated state files:

```bash
bash ~/.local/share/backup/restore/bootstrap.sh apply-state
```

## Snapshot selection

List snapshots first if needed:

```bash
set -a
source ~/.config/restic/env
set +a

restic -r "$(sed -n 's/^repository = "\(.*\)"/\1/p' ~/.local/share/backup/restic/profiles.toml | head -n1)" \
	--password-file ~/.config/restic/password \
	snapshots
```

Use the snapshot ID in `full-restore <snapshot-id>`.

## What apply-state consumes

Expected restored state files:

- `~/.local/state/backup/layered-packages.txt`
- `~/.local/state/backup/flatpaks.txt`
- `~/.local/state/backup/vscode-extensions.txt`

These files are generated by backup hooks and included in restic snapshots.

## Post-restore

- Reboot if layered packages were installed via `rpm-ostree`.
- Verify important apps and configs.
- Re-enable schedules if needed:

```bash
resticprofile -c ~/.local/share/backup/restic/profiles.toml schedule --all --start --reload
```
